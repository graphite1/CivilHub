# CIVIL HUB

施工体制台帳作成・添付書類チェックシステムです。

工事情報、施工体制の業者階層、必要添付書類、添付ファイルを SQLite で管理し、施工体制台帳 Excel からの工事情報取込と、取込元台帳への工事情報反映コピー作成を行います。

## 主な機能

- 工事の登録、編集、削除、一覧表示
- 工事情報 CSV の取込
- 施工体制台帳 Excel からの工事情報取込
- 取込済み施工体制台帳への工事情報反映コピー作成
- 元請、一次下請、二次下請、三次下請以降の業者階層管理
- 登録済み会社情報の編集
- 業者ごとの必要添付書類の自動生成
- 親子関係単位の施工体制台帳・再下請負通知書の管理
- 契約関係帳票の取込元差し替え、登録削除、添付登録削除
- 書類状態、有効期限、備考の更新
- 添付ファイルの登録、保存場所表示、ファイル表示
- 不足、未確認、期限切れ書類の一覧表示
- 添付ファイルのリネーム候補表示

## 必要環境

- Python 3
- tkinter
- openpyxl

`tkinter` は Python の配布形態によって同梱状況が異なります。

## 起動方法

```powershell
python app.py
```

起動時に以下を使用します。

- データベース: `civilhub.db`
- 添付ファイル初期保存先: `storage/attachments`
- 受け渡しフォルダ: `handoff`
- 出力フォルダ: `exports`

## 基本操作の流れ

1. 左ペインの `+ 工事登録` または `工事情報Excel取込` で工事を登録します。
2. 工事一覧から対象工事を選択します。
3. 中央ペインの `+ 元請` で元請業者を登録します。
4. 元請または下請業者を選択し、`+ 子業者` で下位業者を登録します。
5. 会社情報を修正する場合は、施工体制ツリーで会社を選択して `会社情報編集` を実行します。
6. 施工体制ツリーで業者を選択し、下部の `添付書類チェック` で業者単位の必要書類を確認します。
7. 書類を選択して `状態更新` で状態、有効期限、備考を更新します。
8. 書類を選択して `ファイル添付` タブの `+ 添付` からファイルを登録します。
9. `契約関係帳票` タブで、選択中の子業者に対する親子関係単位の帳票を取り込み、反映コピーや添付を管理します。
10. `不足一覧` で、不足・未確認・期限切れの書類を確認します。

## 画面の見方

- 左ペイン: 工事一覧と工事操作
- 中央ペイン: 施工体制ツリー
- 右ペイン: 選択業者の詳細
- 下部タブ: 必要添付書類チェック、添付ファイル一覧、契約関係帳票

工事が未登録の場合は、工事一覧に開始案内を表示します。
工事選択後に業者が未登録の場合は、施工体制ツリーに `+ 元請` から登録する案内を表示します。

## 管理単位

CIVIL HUB では、資料と帳票を次の2階建てで管理します。

- 業者単位の確認資料: 建設業許可証、主任技術者資格証、社会保険加入確認資料、労働保険関係資料、作業員名簿など
- 親子関係単位の帳票: 施工体制台帳、再下請負通知書

親子関係単位の帳票種別は、施工体制ツリーの親子関係から決まります。

- 元請 `level=0` から一次下請 `level=1`: `施工体制台帳`
- 一次下請以降の親子関係: `再下請負通知書`

## CSV取込

工事情報 CSV は次の列名に対応します。

| CSV列名 | 取込先 |
| --- | --- |
| 工事名 | 工事名 |
| 工事番号 | 工事番号 |
| 発注者 | 発注者 |
| 工期開始日 | 工期開始日 |
| 工期終了日 | 工期終了日 |
| 現場代理人 | 現場代理人 |
| 監理技術者 | 監理技術者 |
| 主任技術者 | 主任技術者 |
| 保存フォルダ | 保存フォルダ |

文字コードは `utf-8-sig`、`cp932`、`utf-8` の順に読み取りを試行します。

`project_name` と `excel_output_dir` を含む設定 CSV にも対応し、工事番号は `CFG_` 付きの仮番号として生成します。

## 施工体制台帳 Excel 取込

`工事情報Excel取込` では、`handoff/施工体制台帳_セル対応表.xlsx` の `ユーザー確認` が `OK` の行を使い、選択した `.xlsx` から読み取れる項目を取得します。

取込済みの工事は工事名で照合し、既存工事があれば更新、なければ新規登録します。取込元ファイルとシート名は取込履歴として保存されます。

既存の工事情報項目に対応できる工事名、発注者、工期、現場代理人、監理技術者、主任技術者は工事情報へ反映します。その他の取得値は、項目名、アプリ項目名、セル範囲、値の一覧として内部保持し、取込完了時の確認ダイアログで一部を表示します。

今後の課題:

- 現在の工事情報Excel取込はDBや画面へ反映する項目がまだ少ないため、取得済みの工事情報と会社情報をどこへ保存・表示するか段階的に拡張する。
- 施工体制台帳から元請情報を取得できた場合は、その情報を使って施工体制ツリーへ元請を自動追加する運用を今後検討する。

## 台帳反映

`工事情報反映コピー` は、選択中の工事に対して実行できます。

施工体制台帳の出力では、`handoff/施工体制台帳_マスター.xlsx` をコピーし、`exports` 配下へHub管理版として保存します。マスター本体は上書きしません。

読み書きするセルは `handoff/施工体制台帳_セル対応表.xlsx` の `ユーザー確認` が `OK` の行だけです。`固定で残す` の行は読み書きせず、マスターの値を保持します。

`工事情報反映コピー` は、CIVIL HUBで管理している工事情報を、施工体制台帳マスターのコピーへ反映する操作です。実行前に、マスターを変更しないことと出力予定ファイルを確認します。

## 会社情報編集

施工体制ツリーで会社を選択し、`会社情報編集` から登録済み会社情報を更新できます。

編集対象は、会社名、会社名カナ、代表者名、所在地、電話番号、建設業許可番号、許可有効期限、担当工種、契約日、施工予定日、主任技術者、安全衛生責任者です。

会社名を変更しても、既存の業者単位資料や添付情報は company_id に紐づいたまま維持します。既存の保存先フォルダ名や保存済みファイル名は自動リネームしません。

## 契約関係帳票

`契約関係帳票` タブでは、施工体制ツリーで選択中の子業者から `親会社 -> 子会社` の関係を特定します。

契約関係帳票の `帳票取込` は、選択した Excel の先頭シートを対象に、親子関係単位の帳票ファイルとして紐づけます。複数シートがあっても、初期実装では複数関係とは解釈しません。

`反映コピー作成` は、契約関係帳票が `施工体制台帳` の場合、施工体制台帳マスターをコピーして `exports` へ保存します。`再下請負通知書` は初期実装では取込元 Excel を直接変更せず、同じフォルダに `_civilhub` を付けたコピーを作成します。

契約関係帳票では、帳票情報編集、取込元差し替え、登録削除、帳票添付、添付登録削除ができます。

削除操作はCIVIL HUB上の登録情報だけを削除します。元のExcel/PDFファイルや保存済みファイルの実体は削除しません。

契約関係帳票の添付ファイルは、業者単位の添付とは別に管理します。

## マスター帳票運用方針

将来的には、完成済み帳票から抽出した情報、またはCIVIL HUB上で手入力・編集した工事情報と会社情報を、マスター帳票へ記帳してHub管理版として保存する方針です。

想定するマスター配置:

```text
handoff/
  施工体制台帳_マスター.xlsx
  施工体制台帳_セル対応表.xlsx
templates/
  再下請負通知書_マスター.xlsx
```

施工体制台帳では、左側に工事情報と元請基本情報、右側に選択中の一次下請情報を出力する設計とします。再下請負通知書では、親側に親業者情報、子側に子業者情報を出力する設計とします。

今回の実装では、マスター帳票への全セル出力は未実装です。

## 添付ファイル

添付ファイルは、選択中の工事、業者、必要書類に紐づけて登録します。

保存フォルダが工事に設定されている場合はその配下に保存します。未設定の場合は `storage/attachments/project_<工事ID>` 配下に保存します。

保存先の形式:

```text
<工事保存フォルダまたはstorage/attachments/project_ID>/CIVIL_HUB_添付/<会社名>/<書類名>/
```

保存ファイル名は次の形式です。

```text
<会社名>_<書類名>_v<番号><拡張子>
```

添付登録後、対象書類の状態は `添付済み` に更新されます。

## 開発用初期化

画面右下の `開発用初期化` は、登録済みの工事、業者、必要書類、添付情報、取込履歴を削除する開発用操作です。

通常運用では使用しません。実行時は二段階の確認を行います。
