# CIVIL HUB 操作画面リファレンス

このファイルは、会話で共有された操作画面イメージをもとに、実装時に参照するためのUIメモを保存する。

## 参照画像

- 保存先: `assets/ui-reference/civilhub-operation-screen-reference.png`
- 補足: 現時点では会話添付画像を直接ファイル保存できないため、同名ファイルを後から配置する前提とする

## 画面全体構成

- Windowsデスクトップアプリ風の3カラム構成
- 上部にタイトルバー兼グローバル操作エリア
- 中央に提出書類台帳のメインテーブル
- 左に工事一覧
- 右に書類詳細と添付不足チェック
- 下部に添付ファイルエリア
- 最下部に大型のタブナビゲーション

## ヘッダー

- 左上にハンバーガーメニュー
- アプリ名: `CIVIL HUB`
- サブタイトル: `施工体制・提出管理システム`
- 中央付近に全体検索ボックス
- 右上に通知、設定、ウィンドウ操作アイコン

## 左カラム: 工事一覧

- セクション名: `工事一覧`
- `+ 新規工事` ボタン
- 工事名・工事番号で検索できる検索ボックス
- 工事カードを縦並びで表示
- 選択中カードは青系の強調表示
- 工事カード表示項目:
  - 工事名
  - 工事番号
  - 発注者
  - 工期
- 下部に `工事を開く` ボタン

## 中央カラム: 提出書類台帳

- セクション名: `提出書類台帳`
- 工事件名を括弧付きで表示
- 上部アクション:
  - `+ 新規書類`
  - `編集`
  - `削除`
  - `エクスポート (Excel)`
  - `印刷`
- 右側に書類名検索ボックス
- `フィルター` ボタン

## 書類一覧テーブル

- 主な列:
  - `No.`
  - `種別`
  - `書類名 / 内容`
  - `状態`
  - `提出日`
  - `最新版`
  - `メモ`
- 状態は色付きラベルで表現
- 状態例:
  - `提出済み`
  - `承認済み`
  - `差戻し`
  - `提出待ち`
  - `作成中`
  - `未作成`
- 行選択に応じて右カラムと下部エリアが連動する想定

## 右上: 書類詳細

- セクション名: `書類詳細`
- 入力項目:
  - `種別`
  - `書類名 / 内容`
  - `状態`
  - `提出日`
  - `最新版`
  - `次回提出予定`
  - `メモ`
- 下部ボタン:
  - `保存`
  - `キャンセル`

## 下部中央: 添付ファイル

- セクション名: `添付ファイル`
- 左側にドラッグ&ドロップ領域
- 説明文:
  - `ファイルをドラッグ＆ドロップ`
  - `または`
  - `ここをクリックしてファイルを選択`
- 上部アクション:
  - `+ ファイル追加`
  - `開く`
  - `フォルダを開く`
  - `最新版に設定`
  - `削除`
- 添付一覧の主な列:
  - `ファイル名`
  - `種類`
  - `更新日`
  - `サイズ`
  - `最新版`

## 右下: 添付不足チェックリスト

- セクション名: `添付不足チェックリスト`
- 書類種別に応じてテンプレート切替
- 上部に `テンプレート選択` と更新ボタン
- 主な列:
  - `項目`
  - `必須`
  - `状態`
  - `備考`
- 状態例:
  - `完了`
  - `不足`
  - `未確認`

## フッター・ナビゲーション

- 最下部に大きめのアイコン付きナビゲーション
- タブ候補:
  - `ホーム`
  - `工事管理`
  - `提出書類台帳`
  - `ファイル管理`
  - `チェックリスト`
  - `フォルダ管理`
  - `設定`
- 左下にDB名表示
- 中央付近にユーザー名表示
- 右下にバージョン表示

## デザイン方向

- ダークテーマ基調
- 青系アクセントカラー
- 現場向けの高視認性
- 大型ボタン
- 一覧重視
- Windows業務アプリ風
- Excelライクな情報密度

## 実装時の優先ポイント

- Phase1では左カラムの工事一覧を最優先
- Phase2では中央の提出書類台帳と右の書類詳細を優先
- Phase3では下部の添付ファイルエリアを優先
- チェックリストと下部ナビゲーションは段階実装でよい
