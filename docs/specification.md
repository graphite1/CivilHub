# CIVIL HUB 仕様書

## 目的

CIVIL HUB は、建設工事における施工体制台帳作成と添付書類確認を支援する Windows デスクトップアプリ風の業務アプリケーションである。

工事、施工体制の業者階層、必要添付書類、添付ファイルを一元管理し、施工体制台帳 Excel からの工事情報取込と反映コピー作成を行う。

## 対象範囲

### 対象

- 工事情報の管理
- 施工体制の業者階層管理
- 登録済み会社情報の編集
- 業者ごとの必要添付書類チェック
- 親子関係ごとの施工体制台帳・再下請負通知書管理
- 添付ファイルの登録と保存
- 不足、未確認、期限切れ書類の確認
- 工事情報 CSV 取込
- 施工体制台帳 Excel 取込
- 施工体制台帳 Excel への工事情報反映コピー作成
- 親子関係単位の契約関係帳票 Excel 取込
- 親子関係単位の契約関係帳票反映コピー作成
- 親子関係単位の帳票登録削除、取込元差し替え、添付登録削除

### 対象外

- 添付ファイル内容の自動解析
- 帳票全項目の自動反映
- 複数ユーザー同時編集
- 権限管理
- ネットワーク同期

## 会社情報編集仕様

施工体制ツリーで会社を選択し、`会社情報編集` から登録済み会社情報を編集できる。

対象項目:

- 会社名
- 会社名カナ
- 代表者名
- 所在地
- 電話番号
- 建設業許可番号
- 許可有効期限
- 担当工種
- 契約日
- 施工開始予定日
- 施工終了予定日
- 主任技術者名
- 主任技術者資格
- 安全衛生責任者

保存後の処理:

- `companies` を更新する。
- 施工体制ツリーを再読込する。
- 右ペインの業者詳細を再表示する。
- 業者単位の必要添付書類を再表示する。
- 契約関係帳票を再表示する。

会社名を変更しても、既存の業者単位資料、添付情報、契約関係帳票は ID で紐づけたまま維持する。
既存の保存先フォルダ名や保存済みファイル名は自動リネームしない。

## CSV取込仕様

### 標準CSV

対応列:

| CSV列名 | 保存先 |
| --- | --- |
| 工事名 | projects.name |
| 工事番号 | projects.construction_no |
| 発注者 | projects.client_name |
| 工期開始日 | projects.start_date |
| 工期終了日 | projects.end_date |
| 現場代理人 | projects.site_agent |
| 監理技術者 | projects.managing_engineer |
| 主任技術者 | projects.chief_engineer |

取込条件:

- 工事名と工事番号が空の行は取り込まない。
- 工事番号で既存工事を照合する。
- 既存工事がある場合は更新する。
- 既存工事がない場合は新規登録する。
- 取込前に確認ダイアログを表示する。

### 設定CSV

`project_name` と `excel_output_dir` を含む CSV は設定 CSV として扱う。

| CSV列名 | 保存先 |
| --- | --- |
| project_name | projects.name |
| excel_output_dir | projects.base_folder |

工事番号は `CFG_<工事名>` 形式で仮生成する。

## 工事情報 Excel 取込仕様

対象:

- `.xlsx`
- `handoff/施工体制台帳_セル対応表.xlsx` の `ユーザー確認` が `OK` のセル

読取方針:

- `セル対応表` シートの `代表セル` から読み取る。
- `ユーザー確認` が `OK` の行だけ読み取る。
- `ユーザー確認` が `固定で残す` の行は読み取らない。
- 結合セルは `代表セル` を左上セルとして読み取る。
- 指定シートが取込元Excelに存在しない場合は、先頭シートへフォールバックする。
- 取得値は、項目名、アプリ項目名、セル範囲、代表セル、値の一覧として内部保持する。
- 既存DB項目に対応できる値だけ保存する。
- 日本道路などの完成品台帳はテスト用サンプルとして扱い、共通マスター値や初期データとして使わない。

日付は `YYYY年M月D日` 形式の場合、`YYYY-MM-DD` に変換する。

照合:

- 工事名で既存工事を照合する。
- 既存工事がある場合は更新する。
- 既存工事がない場合は新規登録する。
- 取込元ファイルパスとシート名を `project_imports` に保存する。
- 取込完了時に、取得対象件数、値あり件数、取得値の一部を確認ダイアログで表示する。

今後の課題:

- 現行では取得値のうちDBや画面へ反映する項目が限られるため、取得済みの工事情報、元請基本情報、会社情報の保存先と表示先を段階的に拡張する。
- 施工体制台帳から元請情報を取得できた場合は、工事情報の更新だけで終わらせず、施工体制ツリーへ `level=0` の元請会社を自動追加する仕様を検討する。
- 元請自動追加を行う場合は、既存の元請会社との重複判定、手入力済み会社情報との優先順位、再取込時の更新ルールを整理してから実装する。

## 工事情報反映コピー仕様

実行条件:

- 工事が選択されている。
- `handoff/施工体制台帳_マスター.xlsx` が存在する。
- `handoff/施工体制台帳_セル対応表.xlsx` が存在する。

出力:

- 施工体制台帳マスターは変更しない。
- `exports` 配下にHub管理版を別名保存する。
- ファイル名は `施工体制台帳_<工事名>_<選択会社名>_<YYYYMMDD_HHMMSS>.xlsx` を基本とする。
- 選択会社がない場合は会社名部分を省略する。

セル対応:

- `handoff/施工体制台帳_セル対応表.xlsx` の `セル対応表` シートを使用する。
- `ユーザー確認` が `OK` の行だけを取込・出力対象にする。
- `ユーザー確認` が `固定で残す` の行は読み書きせず、マスターの値を保持する。
- 書込先は `代表セル` とする。
- 結合セルは左上セルである `代表セル` に書き込む。

日付が `YYYY-MM-DD` の場合、開始日は `自 YYYY年M月D日`、終了日は `至 YYYY年M月D日` に変換する。

## 契約関係帳票仕様

契約関係帳票は、工事単位ではなく、施工体制ツリーの `parent_company_id -> child_company_id` の親子関係単位で管理する。

既存の `required_documents` と `attachments` は、建設業許可証、資格証、保険資料、作業員名簿などの業者単位確認資料として維持する。施工体制台帳・再下請負通知書は `company_relation_documents` と `relation_document_attachments` で別枠管理する。

取込条件:

- 施工体制ツリーで子業者が選択されている。
- 選択中の子業者に親業者が存在する。
- 対象ファイルは `.xlsx` とする。
- 複数シートがあっても、初期実装では先頭シートのみを対象にする。

反映コピー:

- 帳票種別が `施工体制台帳` の場合、`handoff/施工体制台帳_マスター.xlsx` をコピーし、セル対応表の `OK` 行だけへ書き込んで `exports` に保存する。
- 帳票種別が `施工体制台帳` の場合、マスター本体は変更しない。
- 帳票種別が `施工体制台帳` の場合、`固定で残す` 行は読み書きせず、マスターの値を保持する。
- 帳票種別が `再下請負通知書` の場合、初期実装ではコピー作成のみを行う。
- 帳票種別が `再下請負通知書` の場合、取込元Excelは直接変更しない。

保存先:

```text
<工事保存フォルダまたはstorage/attachments/project_ID>/CIVIL_HUB_関係帳票/<親会社>__<子会社>/<帳票種別>/
```

### 帳票情報編集

契約関係帳票の状態と備考を編集できる。

編集対象:

- 状態
- 備考

### 取込元差し替え

取込元Excelを差し替える場合、新しい `.xlsx` を選択し、`company_relation_documents` の `source_path`、`source_sheet`、`updated_at` を更新する。

古い取込元Excelファイルは削除しない。

### 帳票登録削除

契約関係帳票の登録削除では、親子関係そのものは必要帳票として残し、取込元・反映コピー・備考・添付登録をクリアして `未確認` に戻す。

削除時の方針:

- 実ファイルは削除しない。
- `source_path`、`source_sheet`、`reflected_path`、`note` を空にする。
- `relation_document_attachments` の登録情報を削除する。
- 削除前に確認ダイアログを表示する。

### 帳票添付登録削除

契約関係帳票の添付登録削除では、`relation_document_attachments` の対象行だけを削除する。

保存済みファイルの実体は削除しない。

## マスター帳票運用方針

CIVIL HUBでは、完成済み帳票から抽出した情報、またはHub上で手入力・編集した情報を、マスター帳票へ記帳してHub管理版として保存する方針とする。

入力ルート:

- 完成済みExcelをHubに入れる -> データ抽出 -> DB保存 -> マスター帳票へ反映 -> 出力
- Hub上で会社情報を手入力・編集 -> DB保存 -> マスター帳票へ反映 -> 出力

想定フォルダ:

```text
handoff/
  施工体制台帳_マスター.xlsx
  施工体制台帳_セル対応表.xlsx
templates/
  再下請負通知書_マスター.xlsx
```

施工体制台帳の出力方針:

- 左側: 工事情報 + 元請情報
- 右側: 選択中の一次下請情報

再下請負通知書の出力方針:

- 親側: 親業者情報
- 子側: 子業者情報

将来の元請基本情報保存案:

- `project_owner_profiles` のような別テーブルで、既存 `projects` にない元請基本情報を保持する。
- 既存 `projects` の工事名、発注者、工期、現場代理人、監理技術者などは引き続き活用する。
- 追加候補は、元請企業名・事業者ID、発注者住所、契約日、発注者監督員、監督員、権限及び意見申出方法、資格内容、営業所、保険番号、外国人就労関連など。

今回の実装では、マスター帳票への全セル出力と `project_owner_profiles` のDB追加は行わない。

## 添付ファイル仕様

登録条件:

- 工事、業者、書類が選択されている。

選択可能ファイル:

- `.pdf`
- `.xlsx`
- `.xls`
- `.docx`
- `.doc`
- `.png`
- `.jpg`
- `.jpeg`
- `.bmp`
- その他すべてのファイル

保存先:

```text
<工事保存フォルダまたはstorage/attachments/project_ID>/CIVIL_HUB_添付/<会社名>/<書類名>/
```

保存ファイル名:

```text
<会社名>_<書類名>_v<番号><拡張子>
```

登録後処理:

- ファイルを保存先へコピーする。
- `attachments` に登録する。
- 対象書類の状態を `添付済み` に更新する。

## 不足一覧仕様

選択中の工事に紐づく書類のうち、次の状態を一覧表示する。

- `不足`
- `未確認`
- `期限切れ`

表示形式:

```text
<会社名>：<書類名> が<状態>
```

## 削除・初期化仕様

### 工事削除

工事削除時、関連する業者、必要書類、添付情報、取込履歴も削除する。

添付ファイルの実ファイル削除は行わない。

### 初期化

デバッグ用初期化として、次のデータを全削除する。

- 工事
- 業者
- 必要書類
- 添付情報
- 取込履歴
- 契約関係帳票
- 契約関係帳票添付情報

`storage/attachments` 配下も削除して再作成する。
